<h1> Introduction to Git </h1>

```console
git add . # using -A will add all changed files to tracking 
git commit -m "" # commit files
git status # check the status of changes in the repo
git reset <filename> # use to unstage a file you don't want committed
git config --list # list configs of the git repo
git push origin master # pushes it to remote repo
git remote add origin <git sshurl> # specifies upstream remote repo
git remote remove origin <git sshurl> # removes upstream repo
git init # initilize a repository
git restore --staged <name-of-the-file> # moves a file from staging to working area
git --version # version of git installed 
echo "notes.txt" >> .gitignore #permanently ignores a personal file
git checkout -b <name-of-the-branch> # creates a branch and immediately switch to it
git push origin <name-of-the-branch> # able to visualize the branch on the GUI
git push origin --delete branch-name # Delete a branch from the remote repo
git log # to see all commits in a Git repo 
git log --oneline # compact, readable
git log --name-only # show each commit plus list of files changed in that commit 
git checkout <commit-hash> # shifts the HEAD back to the specified commit hash
git checkout main/master # goes back to a HEAD state
git branch -M main # changes branch name to main
git branch <branch-name> # creates a new branch
git checkout <branch-name> # switch to a branch 
git push --set-upstream origin <name-of-the-branch> OR
git push -u origin feature-branch # publish the local branch into remote branch
git pull # fetches changes from remote repo and merges them into local repo
```
